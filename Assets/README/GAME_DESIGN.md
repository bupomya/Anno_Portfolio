# Game Design

> 게임의 핵심 기획을 정리합니다.
> 기능 구현, 레벨 디자인, 연출 작업 시 이 문서를 기준으로 일관성을 유지합니다.

---

## 1. 게임 개요

- **장르**: 2D 사이드뷰 플랫포머 (스토리 + 플레이어 조작이 메인)
- **플레이어 캐릭터**: 마법사
- **차별점**: 기존 플랫포머와 다른 마법 사용 메커니즘

---

## 2. 스토리

### 큰 틀
- 마법은 잘 알지만 **감정에 대해서는 잘 모르는 마법사**
- 1년간 여행을 떠남
- 각 계절에 맞는 감정을 몸으로 체험
- 여행을 마치고 출발지로 돌아왔을 때, 출발 시에는 몰랐던 감정을 경험을 통해 이해하게 됨

### 구조적 키워드
- **계절**(봄/여름/가을/겨울) = 챕터 단위
- **감정 성장**
- **원점 회귀** (성장한 상태로)

---

## 3. 계절별 감정 마법 메커니즘

각 계절은 하나의 챕터이며, 고유한 감정과 마법 메커니즘을 가진다.

| 계절 | 감정 | 마법 이름 | 마법 메커니즘 | 플레이 방식 변화 |
|------|------|----------|--------------|-----------------|
| 봄 | 설렘/성장 | Growth (생명력 부여) | 무생물(돌, 나무통)에 잠시 생명을 불어넣어 움직이게 하거나 발판이 되게 함 | 정적인 맵을 동적으로 변화시켜 길을 개척 |
| 여름 | 열정/분노 | Impulse (추진/폭발) | 마법을 바닥이나 벽에 쏘아 그 반동으로 초고속 대시나 2단 점프 수행 (직접 공격보다 이동에 초점) | 스피디하고 박진감 넘치는 액션, 위험 지역을 빠르게 돌파 |
| 가을 | 공허/슬픔 | Phasing (무게 조절/투과) | 자신의 몸이나 물체의 무게를 가볍게 만들어 서서히 낙하하거나 특정 벽을 통과 | 퍼즐 요소 강화, 정밀한 낙하 컨트롤 필요 |
| 겨울 | 고독/냉철 | Stasis (시간 정지/결정화) | 움직이는 플랫폼이나 낙하물을 얼려 일시 정지시키거나 공기 중 수분을 얼려 임시 발판 생성 | 신중한 타이밍 계산, 조작의 정확성 요구 |

---

## 4. 설계 방향

- **Strategy 패턴**으로 계절별 마법을 구현
- 각 마법을 `IMovementBehavior` 구현체로 분리
- 계절 전환 시 전략 객체 교체만으로 조작 변화 표현
- 모든 시스템(마법, 레벨 디자인, 대화, 연출)은 이 기획 방향에 맞춰야 함
